# Tulsi Tales Product Entry Standard

Use this document whenever new products are added to Tulsi Tales. The goal is to make every product publish-ready in one pass, with consistent SEO, clean storefront behavior, and minimal cleanup later.

## 1. Non-Negotiable Goals

Every product must be entered with:
- correct title and clean handle
- correct vendor and product type
- correct variant structure and pricing
- complete, non-generic description and usage copy
- complete metafields
- search-friendly tags
- white-background, correctly padded imagery
- final storefront QA before calling it done

Do not publish products that still need basic cleanup.

## 2. Title Rules

### Product title
Use the clean product name only.

Examples:
- `Dhanwantharam Oil`
- `Brahmi Thailam`
- `Thyrocalm Tablets`
- `Thrunapanchamooladi Kashayam`

### Do not put size in the title
Size belongs in variants, not in the product title.

Wrong:
- `Dhanwantharam Oil 200 ml`
- `Dhanwantharam Oil 450 ml`

Right:
- Product: `Dhanwantharam Oil`
- Variants: `200 ml`, `450 ml`

## 3. Handle Rules

Handles must be clean, readable, and size-free.

Examples:
- `dhanwantharam-oil`
- `brahmi-thailam`
- `thyrocalm-tablets`

Avoid ugly handles like:
- `product-name-200-ml`
- `product-name-1`

If Shopify forces a `-1` suffix because an older draft is occupying the clean handle, check for duplicate draft products first.

## 4. Vendor Rules

Set vendor exactly as the brand name.

Example:
- `Vaidyaratnam`

Do not use mixed capitalization or spelling variants.

## 5. Product Type Rules

Use product types that support search and filtering.

Recommended types:
- Oils and thailams → `Ayurvedic Oils`
- Tablets → `Ayurvedic Tablets`
- Kashayams → `Ayurvedic Kashayams`
- Granules → `Ayurvedic Granules`
- Syrups/liquids → `Ayurvedic Syrups`

Do not leave all products under a vague shared type like only `Ayurvedic Medicine` if a better form-specific type is known.

## 6. Variant Rules

### When to use variants
Use variants whenever the same medicine exists in different pack sizes.

Examples:
- `200 ml`, `450 ml`
- `100 Tablets`
- `10 ml`

### Variant label format
Use clean customer-facing labels.

Examples:
- `200 ml`
- `450 ml`
- `100 Tablets`
- `250 gms`

Do not use:
- `10 nos`
- `TABS`
- inconsistent capitalization

### Variant pricing
Every variant price must match its exact pack size.
Never assume all sizes use the same price.

Before publishing, verify:
- selected size changes the displayed PDP price
- selected size carries into cart correctly

## 7. Tagging Standard

Tags should help:
- storefront search
- internal filtering
- SEO relevance
- future collection logic

### Required base tags
Every product should include:
- brand tag, example: `Vaidyaratnam`
- category tag, example: `Ayurvedic`
- broad store tag, example: `Wellness`
- size tag(s), example: `200 ml`, `450 ml`, `100 Tablets`

### Form/search tags
Use searchable form synonyms.

Examples for oils:
- `Oil`
- `Oils`
- `Ayurvedic Oil`
- `Thailam` when applicable
- `Massage Oil`, `Hair Oil`, `Face Oil`, or `Topical Oil` when applicable
- `External Use`

Examples for tablets:
- `Tablets`
- `Ayurvedic Tablets`
- `Oral Supplement`
- `Internal Use`

Examples for kashayams:
- `Kashayam`
- `Kashayams`
- `Ayurvedic Kashayam`
- `Decoction`
- `Internal Use`

Examples for granules/syrups:
- `Granules` or `Syrup`
- `Ayurvedic Granules` or `Ayurvedic Syrup`
- `Internal Use`

### Support tags
Add 1 to 3 use-case tags where appropriate.
Use support-style language, not spammy keyword stuffing.

Examples:
- `Scalp Care`
- `Skin Care`
- `Joint Support`
- `Lower Back Care`
- `Respiratory Support`
- `Urinary Support`
- `Blood Sugar Support`
- `Heart Support`

Do not overload products with dozens of weak or repetitive tags.

## 8. Description Rules

Descriptions must be:
- specific
- readable
- SEO-friendly
- non-repetitive
- aligned with the actual product form and use

### Description structure
Use this pattern:
1. what the product is
2. main support/benefit context
3. brief differentiator or traditional use context

### Keep ingredients out of the main description
Ingredients belong in metafields, not stuffed into body copy.

## 9. Usage and Dosage Rules

### Dosage
Dosage should reflect the source material exactly where available.

### Usage
Do not use generic filler like:
- `Use as directed.`
- `Apply externally.`

Instead, write a practical customer-facing usage line that matches the dosage and product form.

Examples:
- `Apply to the scalp and hair, massage gently, leave on for at least 30 minutes, then wash if desired.`
- `Take 15 ml with 45 ml warm water twice daily, preferably before food, or as directed by an Ayurvedic physician.`
- `Take 1 to 2 tablets twice daily with water, or as directed by an Ayurvedic physician.`

Usage must not contradict dosage.
Example: a kashayam must never have an external-use instruction.

## 10. Metafield Requirements

Each product should be checked for these where relevant:
- benefits
- dietary preference
- ingredients botanical
- ingredients sanskrit
- product type
- expiry
- HSN/HSM code
- indications
- contraindications
- dosage
- usage
- dosha impact
- brand name
- pack size, where useful

If a field is unknown, leave it intentionally blank or use an approved fallback standard, not random filler.

## 11. Image Rules

All product images must:
- sit on a white background
- be padded to the required card ratio
- avoid awkward zoom/cropping
- be clean and consistent across the catalog

Current preferred standard:
- white background
- padded to 4:5
- no transparency artifacts

## 12. SEO Rules

Before publishing, check:
- clean title
- clean handle
- form-specific product type
- relevant search tags
- clear first paragraph in description
- correct brand spelling
- correct size variants

For oils specifically, include searchable terms so queries like `oil` and `oils` work.

## 13. Publish Checklist

Before marking a product complete:
- title is clean
- handle is clean
- vendor is correct
- product type is form-specific
- all variant labels are clean
- all variant prices are correct
- product is published to the correct sales channel
- tags are complete
- usage is specific, not generic
- dosage is correct
- ingredients are in metafields
- images are consistent
- PDP price changes when size changes
- search query for the product form returns results

## 14. QA Examples by Form

### Oils / Thailams
Must usually include:
- `Oil` / `Oils`
- `Ayurvedic Oil`
- `External Use`
- form-specific tag like `Hair Oil`, `Massage Oil`, `Face Oil`, `Topical Oil`
- size tags for every variant

### Tablets
Must usually include:
- `Tablets`
- `Ayurvedic Tablets`
- `Internal Use`
- `Oral Supplement`
- pack-size tag like `100 Tablets`

### Kashayams
Must usually include:
- `Kashayam`
- `Kashayams`
- `Ayurvedic Kashayam`
- `Decoction`
- `Internal Use`

## 15. Workflow for Future Uploads

1. Gather source data
2. Normalize title, brand, product form, size, and price
3. Create or merge products by base medicine name
4. Create variants for pack sizes
5. Set correct product type
6. Write description, dosage, and specific usage
7. Populate all metafields
8. Add search-friendly tags
9. Clean and upload images
10. Publish
11. QA the storefront
12. Only then mark the task done

## 16. Things to Avoid

Do not:
- create one product per size when variants are more appropriate
- leave size words in titles/handles
- leave generic usage text in place
- use weak placeholder descriptions
- assign the same price to all sizes without checking
- forget to test the PDP after variant setup
- confirm completion before visual and data QA
